## 프로젝트 개요

대용량 데이터를 안정적으로 처리하기 위한 Spring Batch 기반 배치 애플리케이션입니다.
**Reader-Processor-Writer** 패턴을 엄격하게 따르며, chunk 단위 트랜잭션 관리를 통해 안정성을 확보했습니다.

## AI와 함께한 과정

이 프로젝트는 **Claude Code**를 적극 활용하여 개발했습니다.

### 설계 단계
- Spring Batch 6의 변경된 패키지 구조 파악 (Spring Batch 5 → 6 마이그레이션)
- `JobLauncher` → `JobOperator` 전환 등 deprecated API 대응

### 구현 단계
- MyBatis 기반 ItemReader/Writer 구현
- chunk 사이즈 최적화 (기본 1000건)
- 에러 핸들링: skip/retry 정책 설정

## 기술적 하이라이트

### Chunk-oriented Processing

```java
@Bean
public Step processStep(JobRepository jobRepository,
                        PlatformTransactionManager txManager) {
    return new StepBuilder("processStep", jobRepository)
        .<InputDto, OutputDto>chunk(1000)
        .transactionManager(txManager)
        .reader(itemReader())
        .processor(itemProcessor())
        .writer(itemWriter())
        .faultTolerant()
        .skip(DataFormatException.class)
        .skipLimit(100)
        .build();
}
```

### Keyset Pagination

대용량 데이터 조회 시 OFFSET 기반 페이징 대신 keyset pagination을 적용하여 성능을 크게 개선했습니다.

- chunk 기반 트랜잭션 관리의 중요성
