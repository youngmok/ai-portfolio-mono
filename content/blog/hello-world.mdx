## 첫 번째 글

안녕하세요! AI와 함께하는 개발 여정을 기록하는 블로그입니다.

이 사이트 자체가 **Claude Code(Opus 4.6)**와의 협업으로 만들어졌습니다.
설계부터 구현, 배포까지 전 과정을 한 세션에서 진행했고, 그 경험을 첫 글로 남깁니다.

## 어떻게 만들어졌나?

### 1단계: 기획 & 설계

Claude Code에게 "AI 포트폴리오 + 블로그 웹사이트"라는 요구사항을 전달했습니다.
함께 논의하며 다음을 결정했습니다:

- **기술 스택**: Next.js 15 + Tailwind CSS v4 + MDX
- **배포 방식**: GitHub Pages 정적 export
- **콘텐츠 관리**: MDX 파일 기반 (gray-matter + next-mdx-remote)
- **7단계 구현 계획**: 초기화 → 레이아웃 → 콘텐츠 시스템 → 페이지 → SEO → 배포

### 2단계: 구현 (Phase 1~6)

계획대로 단계별로 진행했습니다.
Claude Code가 파일을 생성하고, 빌드 에러가 나면 바로 수정하는 과정을 반복했습니다.

실제로 겪은 이슈들:
- **TypeScript 타입 에러**: `React.ComponentProps<"img">`의 `src`가 `string | Blob` 타입이라 `startsWith` 호출 불가 → `typeof src !== "string"` 가드 추가
- **정적 export 호환**: `robots.ts`, `sitemap.ts`가 route handler 형태라 `output: 'export'`와 충돌 → `public/` 정적 파일 + 빌드 스크립트로 전환
- **워크플로우 브랜치**: `create-next-app`이 `master` 브랜치를 생성하는데 workflow는 `main` 트리거 → `master`로 수정

### 3단계: GitHub 배포

GitHub CLI를 설치하고 인증한 뒤, 리포지토리 생성과 push를 진행했습니다.

```bash
# gh CLI 설치
winget install --id GitHub.cli

# 인증 (브라우저 기반)
gh auth login --web --git-protocol https

# workflow 스코프 추가 (Actions 파일 push에 필요)
gh auth refresh -h github.com -s workflow

# 리포지토리 생성 + push
gh repo create ai-portfolio --public --source=. --push
```

GitHub Pages 활성화도 API로 처리했습니다:

```bash
gh api repos/youngmok/ai-portfolio/pages \
  -X POST -f build_type=workflow \
  -f source[branch]=master -f source[path]="/"
```

### 4단계: 이미지 수정

배포 후 확인했더니 프로젝트 썸네일 이미지가 비어있었습니다.
frontmatter에 `.png` 경로를 지정했지만 실제 파일이 없었기 때문입니다.

SVG로 플레이스홀더 이미지를 만들고, frontmatter 확장자를 `.svg`로 수정하여 해결했습니다.

## 왜 이 블로그를 시작했나?

AI 도구들이 빠르게 발전하면서, 개발 방식 자체가 변하고 있습니다.
단순히 "AI가 코드를 짜준다"가 아니라, **함께 설계하고 의사결정하는 파트너**로서의 역할이 커지고 있죠.

이 과정에서 배운 것들을 기록하면:
1. 나중에 비슷한 작업을 할 때 참고할 수 있고
2. 다른 개발자들에게도 도움이 될 수 있고
3. AI 활용 노하우가 축적됩니다

## 배운 점

- **한 세션에 완성 가능**: 기획부터 배포까지 AI와 함께라면 한번에 끝낼 수 있다
- **에러 대응이 핵심**: AI가 코드를 생성해도 빌드 에러는 피할 수 없다. 빠른 피드백 루프가 중요
- **정적 export의 제약**: Next.js의 `output: 'export'`는 route handler, 이미지 최적화 등에 제한이 있으므로 미리 파악해야 한다

## 앞으로의 계획

- 각 프로젝트의 상세한 작업 일지
- AI 프롬프팅 팁과 패턴
- 기술적 깊이가 있는 포스트
